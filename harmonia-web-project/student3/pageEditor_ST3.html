Feedback Form Page

The feedback form page allows users to share their experiences with the website, helping to improve content quality, usability, and overall engagement. The design follows a structured layout that ensures a smooth and intuitive user experience while maintaining accessibility and responsiveness.

Key Features and HTML Elements Used

This page uses a semantic HTML structure to improve clarity and accessibility. The <form> element organizes all input fields, while <label> elements provide clear descriptions.
The form includes:

  • Text Inputs <input type="text"> and <input type="email"> – Collects user details like name and email.
  • Dropdown Menus <select"> – Used for structured responses like age group selection.
  • Radio Buttons <input type="radio"> – Captures user ratings for satisfaction and content quality.
  • Text Areas <textarea> – Allows users to share detailed feedback or suggestions.
  • Buttons <button> – Enables form submission and reset functions, with JavaScript validation to prevent incomplete submissions.

Navigation is included in the <header>, and a structured footer with social media links and additional website resources ensures consistency across the platform.

CSS Styling and User Experience Enhancements

The feedback form is designed with a clean and user-friendly interface, applying modern CSS techniques such as:

  • Responsive Layout – The form adapts to different screen sizes using max-width: 100% and flexible units.
  • Visual Appeal – border-radius box-shadow, and background colors enhance readability and structure.
  • Interactive Elements – Hover effects box-shadow .inputBoxes provide subtle feedback when users interact with input fields.
  • Button Styling – Custom padding and background transitions (#FEF8DD → #FFE7C7 on hover) improve engagement.
  • Form Spacing and Alignment – padding and margin values ensure proper spacing for easy navigation.

JavaScript Validation and Accessibility

JavaScript ensures that users have entered all required information before submitting the form.

  • validateForm() Function – Prevents submission if required fields are left empty, displaying alerts for missing input.
  • confirmReset() Function – Asks for confirmation before resetting the form to prevent accidental data loss.
  • Keyboard Accessibility – Users can navigate through the form using the Tab key.
  • Interactive Elements – Hover effects box-shadow .inputBoxes provide subtle feedback when users interact with input fields.

Team Page

The team page introduces the contributors behind the website, detailing their roles and responsibilities. The design emphasizes clarity, interactivity, and accessibility to ensure a user-friendly experience.

Key Features and Structure

The page is organized with semantic HTML elements to enhance accessibility and navigation. The main components include:

  • Introduction Section – Provides an overview of the team’s purpose and instructions for interacting with the page.
  • Team Member Profiles – Each member has a designated section displaying their name, role, and contributions.
  • Interactive Hover Effects – Users can hover over a team member’s profile to reveal additional details about their responsibilities.
  • Navigation and Footer – The navigation bar provides access to other sections of the website, while the footer contains useful links.

CSS Design and Enhancements

The visual design incorporates modern CSS techniques to enhance user experience:

  • Responsive Layout – The .team class ensures flexibility, allowing the content to adjust based on the screen size. Aesthetic Styling – The border-radius, box-shadow, and background color #EDF9EB contribute to a clean and professional look.
  • Hover and Transition Effects –
    • .teamMember:hover applies a transform effect to slightly lift the card, adding a dynamic feel.
    • .teamMember:hover .memberDetails reveals additional information with a scale effect.
  • Spacing and Readability – Padding and margin values create a balanced layout, preventing overcrowding of elements.

Interactive Features and User Experience

The page is designed to be engaging and informative through various interactive elements:

  • Hover Effects – Users can reveal a team member’s responsibilities by hovering over their profile image. Keyboard Accessibility – The tabindex="0" attribute ensures that users navigating with a keyboard can access team member profiles.
  • Smooth Transitions – The hover effects are complemented by smooth transitions (transition: all 0.3s ease) to create an engaging user experience.

Disease Prevention and Healthcare Access Page

The Disease Prevention and Healthcare Access page aims to deliver informative content on health-related topics while offering an intuitive and visually engaging user experience. It features well-structured content sections, interactive navigation, and responsive design, ensuring accessibility on a variety of devices.

Key Features and Structure

The page is organized with semantic HTML elements to enhance accessibility and navigation. The main components include:

  • Navigation Bar & Header – A fixed navigation bar provides quick access to different sections of the website, ensuring seamless user interaction.
  • Slide-Out Menu – A hamburger menu (menu icon) allows users to access key topics with ease.
  • Content Sections – Information is divided into clearly defined subsections, each addressing different aspects of disease prevention and healthcare access.
  • Image & Caption Alignment – Relevant images are displayed alongside text to improve content engagement and comprehension.
  • Navigation and Footer – The navigation bar provides access to other sections of the website, while the footer contains useful links.

CSS Design and Enhancements

The page's visual appeal and user experience are enhanced through modern CSS techniques, including:

  • Navigation & Menu Styling:
    • The menu bar (#menuBar) remains hidden by default and smoothly expands upon clicking the menu icon (#menuIcon), improving accessibility.
    • The menu items (#menuBar a) are styled with padding, transitions, and background color changes to enhance usability.
    • The menu footer (.menu-footer) ensures branding consistency by including the website logo and editorial information.
  • Content Styling:
    • The .content class ensures a consistent and structured layout, using border-radius, box-shadow, and padding for a clean presentation.
    • Hover Effects – Smooth transitions (transition: all 0.3s ease) highlight content sections upon user interaction.
    • Image Styling (.images) – Ensures images are properly aligned and resized (width: 45%, display: block; margin: auto;) for optimal viewing.
    • Caption Styling (.imageCaption) – Provides concise explanations using centered text with an adjusted font size for readability.
  • Button & Interactive Elements:
    • The menu icon (#menuIcon) is styled for easy accessibility (position: fixed; cursor: pointer; border-radius: 50px; opacity: 0.7;).
    • Subtopics (.subtopic) use a defined background color and border-radius to maintain consistency with the site's theme.
    • Hover Effects for Menu Items – Items change background color (background-color: #FFE7C7;) when hovered over, providing visual feedback to users.

JavaScript Functionality & Interactive Features

To enhance user experience, JavaScript is used to add interactive menu functionality:

  • toggleMenuBar() Function – Expands or collapses the side menu when the menu icon is clicked, ensuring a smooth sliding effect.
  • Click Outside to Close – A global event listener detects clicks outside the menu bar and automatically hides it, improving user navigation.

Adding interactive elements makes the page more lively and easier to use, reducing clutter and allowing for better navigation. These changes will help the page offer an engaging and smooth experience while keeping it accessible for everyone.

Challenges and Lessons Learned

Technical Challenges & Solutions

  • CSS Consistency: Created unified style.css with color variables for standardized appearance
  • Responsive Design: Implemented flexible boxes and percentage-based widths for multi-device compatibility
  • Navigation: Standardized header/footer elements across all pages for intuitive user experience

Design & Collaboration Insights

  • Interactive Elements: Used CSS transitions for engaging hover effects on team member cards
  • Code Integration: Established file naming conventions and clear directory structure to merge work from team members with varying skill levels
  • Accessibility: Implemented semantic HTML tags and proper image alt text

Most Valuable Learnings

  • CSS variables created a cohesive design system
  • Standardized navigation improved usability
  • Proper file organization enabled cross-referencing between contributors' work
  • Client-side form validation enhanced user experience
