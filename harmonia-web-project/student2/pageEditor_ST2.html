Home Page

The Home Page serves as the main landing page, providing an overview of the website.

Key HTML Elements:

  • <header>: Contains the website title and navigation menu.
  • <body>: Contains the main content of the webpage.
  • <nav>: Provides navigation links to other pages.
  • <section>: Divides the page into meaningful sections.
  • <div>: Helps in structuring sections of a webpage.
  • <h1> - <h4>: Contains headings.
  • <p>: Defines a paragraph.
  • <footer>: Displays copyright and contact information.

Key CSS Properties:

  • display: flex;: Used for aligning elements in the navigation bar.
  • background: Defines background properties.
  • hover effects: Applied to buttons for interactivity.
  • margin: Defines space outside elements.
  • border: Adds a solid border.
  • padding: Defines space inside elements.
  • line-height: Controls spacing between lines.
  • top, left, right: Specifies position relative to the viewport.

Design Justification:

The Home Page is designed to provide a clear, structured, and engaging user experience by utilizing well-organized HTML elements and CSS styling techniques. The use of semantic HTML elements that ensures logical content flow and accessibility. Proper heading structures and paragraph elements enhance readability. CSS properties such as flexbox for alignment, background styling for visual appeal, hover effects for interactivity, and spacing properties like margin and padding help create a visually appealing and user-friendly interface. The strategic use of borders, positioning, and line-height adjustments further refines the presentation, making the page more navigable and aesthetically pleasing.

Accessibility Considerations

The Home Page has been designed with accessibility in mind, ensuring a user-friendly experience for all individuals, including those using assistive technologies.

  • Proper document language declaration:Ensures that screen readers interpret the content with the correct pronunciation.
  • Semantic HTML structure: The use of <header>, <nav>, <section>, and <footer> improves navigation and enhances readability for screen readers.
  • Proper heading hierarchy (h1 - h3): Organizes content logically, making it easier for users to navigate and understand.
  • High contrast colors: White text on a darkened background enhances readability for users with visual impairments.
  • Text scaling: The use of relative units (rem) allows text to adjust based on user preferences, improving accessibility.
  • Smooth scrolling (scroll-behavior: smooth): Enhances navigation by providing a seamless transition between sections.
  • Consistent navigation: The placement and behavior of navigation elements remain uniform across pages, improving usability.

These accessibility features ensure that the Home Page is inclusive and provides a seamless experience for all users, regardless of their abilities or devices.

Wellness Table

This page includes a structured table that outlines healthy habits.

Key HTML Elements:

  • <table>: Creates the structured layout.
  • <ul> and <li>: Used for listing benefits in each habit.
  • <th>: Defines table headers.
  • <td>: Holds the data in each row.
  • <tr>: Table rows.
  • <a>: Hyperlinks to different sections or external sources
  • <link>: Links to external CSS files and fonts

Key CSS Properties:

  • border-collapse: collapse;: Merges table borders for a cleaner look.
  • nth-child(even): Used to style alternating rows for better readability.
  • border: Defines borders for tables, rows, and cells
  • font-family: Specifies the font type
  • ::before: Inserts content before an element (✔ checkmark)
  • border-radius: Adds rounded corners to elements

Design Justification:

The Wellness Table was designed to present healthy habits in a structured and visually clear manner. The table layout ensures that information is easy to scan, while alternating row colors enhance readability. Key benefits of each habit are displayed using bullet points, maintaining a clean and organized look. Typography and spacing are carefully applied to highlight important details, ensuring a balanced visual hierarchy. Borders and padding contribute to a well-defined layout, making the content easy to understand at a glance. The use of icons and visual indicators, such as checkmarks, improves engagement and understanding.The table is designed to scale efficiently, maintaining clarity as more wellness habits are added.

Accessibility Considerations

  • HTML structure:The use of <table>, <th>, <tr>, and <td> ensures proper screen reader interpretation.
  • Table headers (<th>):Clearly define column meanings, aiding comprehension for users relying on screen readers.
  • Alternating row colors (nth-child(even)): Improves readability for users with visual impairments.
  • Consistent font styling (font-family Enhances legibility across different devices.
  • Descriptive hyperlinks (<a> elements): Ensure meaningful descriptions.
  • Sufficient contrast in text and background colors: Ensures that users with visual impairments can easily read the content.

Physical Health Content Page

This page provides information on physical health, using text and images.

Key HTML Elements:

  • <nav>: Defines the navigation section.
  • <ul>: Unordered list for navigation links.
  • <a>: Anchor links to jump to sections.
  • <h1> and <h2>: Main heading and Subheadings.
  • <p>: Groups related content.
  • <img>: Displays relevant images.
  • <button>: Scrolls the user back to the top.

Key CSS Properties:

  • min-height: 100vh;: Makes the section take the full viewport height.
  • background-color: rgba(0, 0, 0, 0.5);: Adds a semi-transparent black overlay.
  • linear-gradient(to top, rgba(0, 0, 0, 0.5), transparent), linear-gradient(to left, rgba(0, 0, 0, 0.5), transparent), url("img/img7.webp");: Sets multiple background layers.
  • font-size: Sets text size.
  • text-decoration: none;: Removes underline from links.
  • z-index: 1000;: Ensures navigation stays on top.

Design Justification:

The Physical Health Content Page is designed to provide an engaging and informative experience by balancing text and imagery effectively. The layout is structured to ensure smooth navigation, with a clear hierarchy that guides users through the content. Navigation links allow for easy access to different sections, while anchor links improve the user journey by enabling quick jumps to relevant topics.Typography choices and spacing prioritize readability, while the scroll-to-top button improves usability by allowing users to return to the top effortlessly. With these elements in place, the design remains intuitive, aesthetically appealing, and easy to navigate.

Accessibility Considerations

  • Semantic HTML elements such as <nav>, <h1>, <h2>, <p>, and <img> ensure compatibility with assistive technologies.
  • The semi-transparent overlay and text contrast improve readability for users with visual impairments.
  • Anchor links and a fixed navigation bar enhance keyboard navigation, making it easier for users to move between sections.
  • All images include descriptive alt attributes, allowing screen readers to convey visual content to users.
  • Relative font sizes (rem, em) ensure text scales properly based on user preferences.
  • The z-index property keeps the navigation bar accessible at all times, preventing it from being obscured by other content.
  • Focus indicators should be implemented for interactive elements to assist users relying on keyboard navigation.

Challenges and Lessons Learned

Challenge: Structuring the Wellness Table

One of the major challenges I faced was ensuring that the wellness table remained visually structured and easy to read while maintaining accessibility.

Resolution: I used semantic HTML elements such as <table>, <th>, and <td> to provide a clear and structured layout. Additionally, I applied border-collapse: collapse; and nth-child(even) in CSS to improve readability and make alternating rows visually distinct.

Lesson Learned: Using semantic elements enhances both accessibility and maintainability. I also learned the importance of styling tables properly to prevent cluttered layouts and ensure a clean design.

Challenge: Ensuring Accessibility

One of the challenges I faced was making the wellness table easy to navigate for users with screen readers.

Solution: I added aria-label attributes to the table and interactive elements to provide meaningful descriptions. Additionally, I used proper heading hierarchy (h1-h3) to enhance readability.

Challenge: Creating a Responsive Table

Ensuring the table adapted properly to different screen sizes was difficult, as some content overflowed.

Solution: I wrapped the table inside a <div> with overflow-x: auto; to enable horizontal scrolling on smaller screens. I also used relative units like rem and % to maintain scalability.

Challenge: Maintaining Navigation Consistency

Ensuring the navigation bar remained consistent across pages and accessible while scrolling was another challenge.

Solution: I used position: fixed; and z-index: 1000; to keep the navigation bar visible at all times, preventing it from being hidden behind other elements.

Challenge: Aligning Text and Elements

Positioning certain elements, such as table headers and descriptions, was tricky.

Solution: I used text-align: center; for headers and flexbox for alignment, ensuring a visually balanced layout.
