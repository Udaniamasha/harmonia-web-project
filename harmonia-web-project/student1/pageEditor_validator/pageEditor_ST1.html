Splash Page

Technical Description

The splash screen serves as the initial entry point to our website, designed to create a memorable first impression while effectively communicating our project's purpose focused on health and well-being.

Key HTML Elements Used:

  • <!DOCTYPE html> and <html lang="en">: Defines the document type and language for accessibility.
  • <meta charset="UTF-8">: Ensures proper text encoding.
  • <meta name="viewport">: Enables responsive design for mobile devices.
  • <meta http-equiv="refresh">: Auto-redirects to home.html after 4 seconds.
  • <video> with autoplay muted loop playsinline: Background video element with attributes for continuous, silent playback across devices.
  • <img> with alt attributes: Logo elements with descriptive alternative text.
  • Semantic elements like <h1>, <h2>, <p>, <ol>, <li>: Provides document structure and hierarchy.
  • <div> containers: Used for grouping and styling related content sections.

CSS Techniques Applied:

  • Flexbox layout: display: flex with justify-content and align-items centers content vertically and horizontally.
  • Positioning: position: absolute with inset: 0 for the background and strategic positioning for logos and team section.
  • CSS animations: keyframes for fadeUp, fadeIn, and spin effects creating a staggered entrance sequence.
  • Text styling: background-clip: text with linear gradient creates a sophisticated text effect for the title.
  • Media queries: Responsive adjustments for screens below 768px width.
  • Z-index management: Ensures proper layering of elements with the video background behind content.
  • Filter effects: brightness(0.4) on the video improves text visibility.
  • CSS transitions: Gradual opacity changes for smoother animations.

Design Justification:

These design elements work together to create a visually appealing and user-friendly interface. The dark video overlay enhances readability while maintaining engagement, and the centered content with absolute positioning establishes a clear hierarchy. Staggered animations guide the viewer's focus, while gradient text effects add a touch of sophistication. Circular logos with shadows ensure brand presence without clutter, and a loading spinner provides feedback during redirections. The full viewport layout delivers an immersive experience, with strategic spacing ensuring a clean and balanced design.

Accessibility Considerations:

  • Proper document language declaration: Helps screen readers use correct pronunciation.
  • Semantic HTML structure: Improves navigation for assistive technologies.
  • Alt text for images: Provides descriptions for screen reader users.
  • Color contrast: White text on darkened background ensures readability.
  • Text scaling: Relative units (rem) allow text to scale with user preferences.
  • Video attributes: muted prevents unexpected audio, while playsinline prevents unexpected fullscreen on mobile.
  • Text shadows: Enhances readability over variable background.
  • Responsive design: Ensures usability across different device sizes.
  • Focus states: Not explicitly shown but should be included for keyboard navigation.
  • Meta refresh: Provides automatic redirection but at 4 seconds gives adequate time for users to perceive content.

Voluneteer Page

Technical Description

The volunteer page provides information about health-related volunteer opportunities, presenting them in an organized and visually appealing layout.

Key HTML Elements Used:

Document Structure Tags:

  1. <!DOCTYPE html>: Defines the document type as HTML5.
  2. <html lang="en">: Root element with English language specification.
  3. <head>: Contains meta information, title, and stylesheet links.
  4. <body>: Contains all visible content.

Structural Section Tags:

  1. <header>: Contains the site logo, title, and navigation.
  2. <main>: Holds the primary content (volunteer programs).
  3. <footer>: Contains copyright information and secondary navigation links.

Navigation Elements:

  1. <nav> with <ul> and <li>: Creates the main navigation menu.
  2. <a href="...">: Creates hyperlinks to other pages.

Content Organization:

  1. <div class="container">: Used repeatedly for content wrapping and layout control.
  2. <div class="programs-grid">: Contains the program cards in a grid layout.
  3. <div class="program-card">: Individual program information containers.

Form Elements:

  1. <select> with <option>: Creates a dropdown filter for programs.

Text Formatting:

  1. Various heading levels (<h1>, <h3>): Define headings with different importance.
  2. <p>: Defines paragraphs of text.

CSS Integration:

  1. <link rel="stylesheet">: Links to external CSS file.
  2. Multiple class attributes: Used to apply styling from the CSS file.

CSS Techniques Applied:

  • CSS Variables: :root with custom properties like --primary-color creates a unified color scheme across the site.
  • Flexbox layout: display: flex with justify-content: space-between in header for navigation alignment.
  • CSS Grid: .programs-grid uses grid-template-columns: repeat(2, 1fr) for a responsive two-column card layout.
  • Hover animations: transform: translateY(-5px) and box-shadow changes create card elevation effects.
  • Pseudo-elements: nav ul li a::after builds custom underline animations for navigation links.
  • Sticky positioning: position: sticky; top: 0; keeps the header visible while scrolling.
  • Custom form styling: appearance: none with SVG background for the dropdown selector.
  • Box-shadow effects: box-shadow: var(--shadow) adds depth to cards and UI elements.
  • CSS transitions: transition: var(--transition) creates smooth hover effects throughout.
  • Background images: background-size: cover with fixed height creates consistent card header images.
  • Flexible card layout: display: flex; flex-direction: column; with flex-grow: 1 for equal-height cards.
  • Border-radius: border-radius: 12px with overflow: hidden creates clean rounded corners on cards.

Design Justification:

The volunteer page was designed with a card-based layout to ensure that each opportunity is presented in a structured and visually consistent manner. The design emphasizes readability and accessibility, using typography and color hierarchy to highlight key details. A dropdown menu is included to suggest sorting options for future functionality while maintaining a clean and intuitive user experience. The layout remains effective as the number of volunteer opportunities grows, ensuring scalability and ease of navigation.

Accessibility Considerations:

For the volunteer page, I focused on ensuring accessibility through:

  • Semantic HTML structure: Improves navigation for assistive technologies.
  • Responsive design that adapts to different screen sizes and zoom levels
  • Sufficient color contrast between text and background for readability
  • Descriptive alt text for all images including logo, program images, and profile pictures
  • Fixed navigation header ensuring constant access to main site sections.
  • Back-to-top button reducing scrolling requirements
  • Clear visual distinction between different content sections.
  • Appropriately sized text with adequate line spacing for readability.

Project Pages

Technical Description

The project pages provide detailed information about various volunteer projects, including project descriptions, objectives, target audience, and volunteer roles. They feature ratings and feedback sections where volunteers can share their experiences, along with a user-friendly feedback form that dynamically updates the page upon submission. Each page maintains a consistent structure to ensure ease of navigation and engagement through interactive elements like the feedback form and review expansion.

Key HTML Elements Used:

  • <header>: Contains the navigation menu and branding elements for consistency across pages.
  • <section>: Organizes content like project descriptions, feedback, and ratings into structured blocks.
  • <form>: Enables users to submit feedback dynamically, enhancing interactivity.
  • <button>: Triggers interactions such as submitting feedback and expanding review details.
  • <div class="review-card">: Structures individual user reviews, ensuring a clean and readable layout.
  • <script>: Implements dynamic features like form submission handling and review expansion.

CSS Techniques Applied:

  • Flexbox Layout: display: flex; with justify-content and align-items centers content both vertically and horizontally, ensuring a responsive and clean layout.
  • CSS Animations: @keyframes for fadeIn and fadeOut effects to create smooth transitions for form submissions and page changes.
  • Text Clamping: -webkit-line-clamp: 2; truncates longer reviews, and the "See more" button expands the text, improving readability without overwhelming the page.
  • Positioning: position: absolute; and z-index ensure elements are layered correctly, with video backgrounds placed behind the content.
  • Hover Effects: transform: scale(1.05); and transition: 0.3s; smoothens hover interactions for buttons and cards, enhancing user engagement.
  • Media Queries: Responsive adjustments with @media for different screen sizes, optimizing layout and design for mobile and desktop views.

Design Justification:

The design focuses on a clean, responsive layout using Flexbox for alignment and media queries for mobile optimization. CSS animations and transitions create smooth interactions, while text clamping for reviews maintains a tidy look. Buttons and links are styled with hover effects to enhance user engagement, ensuring a visually appealing and interactive experience across devices.

Accessibility Considerations:

  • Alternative Text for Images: <img src="images/logo.jpg" alt="Harmonia Logo"> ensures screen readers describe the image content.
  • Keyboard & Screen Reader Navigation: Proper use of semantic HTML (<nav>, <header>, <footer>) improves accessibility.
  • ARIA Attributes for Interactive Elements: The "Back to Top" button includes aria-label="Back to top of page" for clarity.
  • Color Contrast & Readability: The color scheme ensures good readability, with a background color (#CAF1DE) for success messages improving visibility.

Application Page

Technical Description

The "Volunteer Application" page collects personal and volunteer-related information using a structured form. It features input validation, dropdowns, and text areas, with a submit button redirecting users to a thank-you page. The design is responsive, ensuring accessibility with semantic HTML and ARIA attributes.

Key HTML Elements Used:

  • <!DOCTYPE html> and <html lang="en">: Defines the document type and language for accessibility.
  • <meta charset="UTF-8">: Ensures proper text encoding.
  • <meta name="viewport">: Enables responsive design for mobile devices.
  • <nav>: Defines navigation links for easy page traversal.
  • <form>: Contains the volunteer application form with various input types like <input>, <select>, and <textarea>.
  • <button>: Submit button for form submission.
  • <header> and <footer>: Structure the header and footer sections for consistency across pages.
  • <a>: Links for navigation and applying to volunteer programs.

CSS Techniques Applied:

  • Flexbox layout: display: flex with justify-content and align-items for centering content in the header and form elements.
  • Positioning: position: relative and absolute for precise placement of the back link, buttons, and footer sections.
  • Responsive design: Media queries adjust the layout for smaller screen sizes to improve mobile usability.
  • Form styling: Input fields, textareas, and select elements are styled for consistency and ease of use with padding, borders, and font size adjustments.
  • Button styling: background-color and hover effects on the submit button for improved user interaction.
  • Typography: Clear font choice with font-family for better readability and user experience.
  • Color contrast: Ensures sufficient contrast between text and background, improving readability, especially for buttons and form labels.

Design Justification:

The page offers a simple, intuitive application process with a clean layout for easy navigation. Semantic HTML improves accessibility, and interactive elements are styled for easy interaction. The design ensures responsiveness across desktop and mobile devices.

Accessibility Considerations:

  • Proper document language declaration: Helps screen readers use correct pronunciation.
  • Semantic HTML structure: Improves navigation for assistive technologies.
  • Alt text for images: Provides descriptions for screen reader users.
  • Form Labels: Each input field has a corresponding label, improving keyboard navigation and screen reader compatibility.
  • Responsive design: Ensures usability across different device sizes.
  • ARIA Labels: Interactive elements, like buttons, have clear ARIA labels for better accessibility.

Challenges and Lessons Learned

Challenge: Responsive Design Implementation

One of the major challenges I faced was ensuring that all pages remained visually appealing and functional across different screen sizes, particularly with the complex layout of the volunteer page.

Resolution: I implemented media queries with careful breakpoints and used flexible layouts with CSS Grid and Flexbox. I tested extensively across different devices and adjusted the design accordingly.

Lesson Learned: Planning for responsiveness from the beginning of the design process is more effective than attempting to retrofit it later. I also learned the importance of using relative units (rem, em, %) rather than fixed pixel values.

Challenge: Maintaining Consistent Styling

Working with a comprehensive CSS stylesheet provided a challenge in understanding and correctly implementing the established design system while making necessary adjustments for specific page needs.

Resolution: I spent time analyzing the CSS variables and class patterns, creating a personal reference guide for the styling system. I also used browser developer tools to test how changes would affect the appearance before implementing them.

Lesson Learned: The importance of well-organized CSS with clear naming conventions and the value of CSS variables for maintaining consistency. I also learned how to effectively extend an existing design system without breaking its core principles.

Challenge: Content Organization on Health Topics

Organizing comprehensive health and well-being information in a way that was both informative and engaging proved challenging, especially with the goal of creating clear connections between different wellness concepts.

Resolution: I created a content hierarchy with main topics and subtopics, using visual cues through CSS to establish relationships. I also incorporated progressive disclosure techniques to avoid overwhelming users with information.

Lesson Learned: The importance of information architecture in web design and how visual design can support content organization. I also gained insight into how to balance comprehensiveness with usability when presenting complex health information.

Compliance

In developing the Splash Screen, Volunteer Page, and Content Page, I ensured compliance with Janet regulations governing web page publication through the following measures:

  • Content Appropriateness: All content was created to be educational and informative about health and well-being, avoiding any inappropriate or offensive material.
  • Intellectual Property: Original content was created for the pages, and any external references are properly cited in accordance with academic standards.
  • Data Protection: The pages do not collect personal information from users without clear consent, and all form fields that might request information include appropriate privacy notices.
  • Technical Standards: The HTML and CSS code follows modern web standards and has been validated using W3C validators to ensure proper functioning across different browsers and devices.
  • Accessibility Compliance: As detailed in the Accessibility section, all pages have been developed with WCAG guidelines in mind to ensure they are usable by people with disabilities.
  • Security Considerations: The pages implement secure coding practices to prevent common vulnerabilities, particularly in any interactive elements.

By adhering to these principles, the pages I developed contribute to a website that is legally compliant, ethically sound, and follows best practices for academic web publication.
