Profile Page

Technical Description

HTML Structure and Organization

Key HTML Elements Used:

  • <!DOCTYPE html> and <html lang="en">: Establishes document type and language for proper rendering and accessibility.
  • <meta charset="UTF-8"> and <meta name="viewport">: Ensures correct character encoding and responsive behavior across devices.
  • <link rel="stylesheet">: Connects to external CSS files including Font Awesome for icons.
  • <section> with aria-labelledby: Creates semantic content blocks with proper accessibility relationships.
  • <h2>, <h3>, <h4>: Establishes clear content hierarchy throughout the document.
  • <ul> and <li>: Organizes related content like skills, goals, and services into structured lists.
  • <table>: Implements the availability calendar with appropriate row and cell structure.
  • <form> with labeled <input> and <textarea>: Creates an accessible contact interface.
  • <div class="grid-container">: Serves as the main wrapper for the entire profile layout.

CSS Techniques Applied

CSS Techniques Applied:

  • CSS Variables: :root variables for colors, shadows, and transitions maintain consistency and simplify theme management.
  • Box Model Normalization: Universal box-sizing: border-box prevents layout inconsistencies across elements.
  • CSS Grid Layout: display: grid with grid-template-columns: repeat(auto-fit, minmax(320px, 1fr)) creates a responsive, automatically adjusting layout.
  • Card Pattern: .card class with consistent styling (border-radius, shadow, padding) establishes visual separation between content sections.
  • Color Theming: Section-specific background colors using rgba values with transparency for visual distinction while maintaining cohesion.
  • Interactive Elements: Hover states with transform: translateY(-3px) and enhanced shadows provide subtle user feedback.
  • Skill Visualization: Gradient-filled progress bars with data attributes display skill proficiency visually.
  • Transitions: transition: var(--transition) ensures smooth animations throughout the interface.
  • Flexbox Layouts: Used within cards for alignment and spacing of content (particularly in the user info section).
  • Vendor Prefixes: -webkit, -moz, and other prefixes ensure cross-browser compatibility.

Accessibility

The profile page implements several key accessibility features:

  • Semantic HTML Structure: Proper use of sectioning elements and headings creates a logical document outline.
  • ARIA Attributes: Each section includes aria-labelledby references connecting to their respective headings.
  • Screen Reader Support: The .sr-only class hides visual elements while keeping them available to screen readers.
  • Image Alternatives: All images include descriptive alt attributes.
  • Form Accessibility: Input fields are properly associated with their labels using for and id attributes.
  • Color Contrast: Dark text colors on light backgrounds ensure sufficient contrast for readability.
  • Focus States: Interactive elements include visible focus states for keyboard navigation.
  • Responsive Design: The layout adapts to different screen sizes through media queries.
  • Text Scaling: Typography uses relative units allowing content to scale with user preferences.
  • Icon Accessibility: Icons are accompanied by text labels, ensuring their meaning is clear to all users.

Sitemap Page

Technical Description

The sitemap page provides an interactive visual representation of the website's structure using SVG graphics, allowing users to navigate the site hierarchy through an intuitive diagram.

Key HTML Elements Used:

  • Document Structure Tags:
    • <!DOCTYPE html>: Defines the document as HTML5.
    • <html lang="en">: Root element with English language specification.
    • <meta name="viewport">: Ensures proper rendering on various devices.
    • <link rel="stylesheet">: Links to normalize.css, custom CSS, and the Inter font.
  • Content Organization:
    • <div class="container">: Main wrapper for all content.
    • <header>: Contains the page title and description.
    • <main>: Holds the primary content.
    • <div class="sitemap-container card">: Contains the SVG sitemap.
    • <div class="sitemap-instructions card">: Contains usage instructions.
  • SVG Implementation:
    • <svg id="sitemap">: The main SVG element containing the sitemap visualization.
    • <g class="node-group">: Groups for interactive nodes with encapsulated elements.
    • <rect class="node">: Rectangle elements representing pages.
    • <text class="node-text">: Text labels for each page node.
    • <line class="connector">: Lines connecting nodes to represent hierarchy.
  • Interactive Elements:
    • <a href="...">: Hyperlinks within each node group for navigation.
    • tabindex="0": Makes nodes keyboard-focusable.
    • role="link": Defines the semantic role for accessibility.
    • title and aria-label attributes: Provide additional context.

CSS Techniques Applied:

  • CSS Variables: Using :root with custom properties for colors, shadows, and transitions creates a cohesive design system.
  • Card Styling: .card class with subtle background color, border-radius, and shadow effects.
  • Interactive Node Styling: Hover and focus effects using transform and box-shadow; color-coded nodes (home, sub-nodes, active node) for visual differentiation.
  • Responsive Design: Media queries adjust font sizes and spacing at different breakpoints; SVG viewBox adjustment via JavaScript for proper rendering at any screen size.
  • Typography Styling: Font family hierarchy with fallbacks; decorative heading underlines using pseudo-elements; size differentiation for hierarchy.

JavaScript Features:

  • SVG viewBox Adjustment: Dynamically calculates and sets the SVG viewBox based on container dimensions for responsive rendering.
  • Keyboard Accessibility: Event listeners for Enter key to activate links when nodes are focused.
  • Resize Handling: Recalculates viewBox on window resize events.

Accessibility

The sitemap implementation addresses accessibility through several key techniques:

  • Semantic Structure:
    • Using proper heading hierarchy (<h1> for page title, <h2> for instructions section).
    • Semantic grouping with <header> and <main> elements.
  • SVG Accessibility:
    • role="link" attributes define the semantic purpose of node groups.
    • tabindex="0" makes all nodes keyboard-navigable.
    • title elements provide tooltips for additional context.
    • aria-label and aria-labelledby attributes offer descriptive information for screen readers.
  • Keyboard Navigation:
    • JavaScript implementation ensures Enter key activates links for keyboard users.
    • Focus styling (matching hover effects) helps identify the currently focused element.
    • Logical tab order follows the visual hierarchy of the sitemap.
  • Visual Clarity:
    • Color-coded nodes with sufficient contrast ratios.
    • Current page highlighted with a distinct style (active-node class).
    • Text size adjustments in media queries maintain readability on smaller screens.
    • Clear instructions provided for using the sitemap.
  • Responsiveness:
    • Dynamic SVG viewBox adjustment ensures the diagram is visible and usable at any screen size.
    • Font size reductions at smaller viewports prevent text overflow.
    • Preserved SVG aspect ratio prevents distortion.

Challenges and Lessons Learned

Developing the Content, Sitemap, and Profile pages presented several challenges, each offering valuable learning opportunities. These challenges spanned across HTML structure, CSS styling, JavaScript implementation, and accessibility considerations.

Challenge 1: Implementing a Responsive and Accessible Sitemap with SVG

Problem: Ensuring the sitemap diagram scaled correctly on all screen sizes, maintained readability, and provided a good user experience for both mouse and keyboard users required careful planning.

Solution:

  • Dynamic viewBox calculation using JavaScript to scale the SVG proportionally.
  • Media queries to adjust font sizes and spacing at different breakpoints.
  • Keyboard navigation enhancements using event listeners for Enter key activation.
  • ARIA attributes for improved accessibility.

Lesson Learned: Planning for accessibility from the beginning is crucial to an inclusive user experience.

Challenge 2: Maintaining Consistent Styling and Layout Across Pages

Problem: Ensuring a consistent look across different pages while adapting to their specific content and functionality.

Solution:

  • CSS variables for colors, shadows, and transitions.
  • Reusable CSS classes for consistent design.
  • Logical organization of CSS files for maintainability.

Lesson Learned: A well-planned CSS architecture ensures scalability and maintainability.

Challenge 3: Creating a Responsive and Accessible Profile Page Layout

Problem: Balancing content sections for readability and accessibility on all screen sizes.

Solution:

  • CSS Grid for flexible layouts.
  • Semantic HTML elements for better structure.
  • ARIA attributes and form accessibility enhancements.
  • Thorough testing across devices and assistive technologies.

Lesson Learned: Modern layout techniques like Grid and Flexbox improve responsiveness and accessibility.

Challenge 4: Implementing Fixed Sidebar and Sticky Positioning

Problem: Overlapping of sticky/fixed elements with page content.

Solution:

  • Using top: 100px to prevent overlap with navigation bars.
  • Adjusting position: fixed settings and z-index to resolve conflicts.

Lesson Learned: Consideration of all page elements prevents layout conflicts.

Challenge 5: In-page Navigation and Responsive Sidebar

Problem: Creating smooth in-page navigation with a responsive sidebar.

Solution:

  • Using fragment identifiers (# links) for smooth scrolling.
  • Applying position: fixed to keep the sidebar visible while scrolling.
  • Media queries for horizontal navigation adaptation on smaller screens.

Lesson Learned: Responsive design must consider multiple screen sizes and user interactions.
